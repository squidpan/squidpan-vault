## âš™ Layout Engines and Options

PlantUML allows you to use several different [layout engines](https://plantuml.com/layout-engines):
- [Graphviz](https://plantuml.com/graphviz-dot) is the default engine. Relies on an external program;
- [Smetana](https://plantuml.com/smetana02). *(Use `!pragma layout smetana` in the file or `-Playout=smetana` on the command-line.)* A port of Graphviz to Java. Tends to make slightly straighter arrows;
- [VizJs](https://plantuml.com/vizjs) uses JavaScript. *(Use `-graphvizdot vizjs` on the command line.)* Tends to space out nodes more, resulting in a bigger diagram;
- [ELK (Eclipse Layout Kernel)](https://plantuml.com/elk). *(Use `!pragma layout elk` in the file or `-Playout=elk` on the command-line.)* Supports only orthogonal layout, and doesn't cover all features.
[Sequence diagrams](https://plantuml.com/sequence-diagram) have 2 engines:
- *Puma* is the older and still default engine;
- [Teoz](https://plantuml.com/teoz) *(Use `!pragma teoz true` in the file or `-Pteoz=true` on the command-line)* is a new engine that has some unique features: *anchors (named arrows/messages), duration, nested boxes...*

> Achieving good layout with PlantUML is sometimes non-trivial. *See [PlantUML GraphViz Layout](https://isgb.otago.ac.nz/infosci/mark.george/Wiki/wiki/PlantUML%20GraphViz%20Layout) by Mark George at University of Otago for more advice.*
